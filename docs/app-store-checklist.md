# 앱스토어 / 플레이스토어 / 앱인토스 심사 체크리스트

심사 직전에 한 번 더 처음부터 끝까지 따라가세요.
모든 항목이 `[x]`가 되어야 제출합니다.

---

## 공통 (iOS / Android / 앱인토스)

- [ ] 앱 이름 / 부제 / 설명에 가챠 확률 페이지 링크 명시
- [ ] 스크린샷 5장 이상 (게임 플레이 / 보상 / 랭킹 / 마이 / 가챠)
- [ ] 14세 이상 등급 표시 + 인앱 결제 / 확률형 아이템 항목 체크
- [ ] 개인정보처리방침 / 이용약관 URL 공개
- [ ] 회원 탈퇴가 마이페이지에서 **2탭 이내** 도달 가능
- [ ] 신고 / 차단 기능 동작 확인
- [ ] 아이콘 1024x1024 (iOS) / 512x512 (Android) 적응형 아이콘 포함

## iOS

- [ ] **Sign in with Apple** 동작 확인 (Apple 정책 4.8)
- [ ] Info.plist의 권한 키들이 실제로 사용 중인 것만 등록 (미사용 시 거절)
  - `NSCameraUsageDescription`, `NSMicrophoneUsageDescription` 등
- [ ] App Tracking Transparency가 필요한 경우 프롬프트 구현
- [ ] Apple 자체 화폐 외 결제 안내 문구 없음 (외부 결제 유도 금지)
- [ ] 가이드라인 5.3 (도박/확률) — 가챠 확률 표시 링크가 앱 내에서도 1탭 이내
- [ ] iPadOS 빌드 비활성 또는 iPad 호환성 검증

## Android (Play 콘솔)

- [ ] `targetSdkVersion` 최신
- [ ] 권한 사용 목적 명시 (확장 정보 양식)
- [ ] 데이터 안전 섹션 정확하게 입력
- [ ] 14세 이용가 + 콘텐츠 등급 신고
- [ ] 로컬라이즈된 메타데이터 (한국어 / 영어 최소)

## 앱인토스 (미니앱)

- [ ] 미니앱 매니페스트 작성 (이름 / 카테고리 / 아이콘)
- [ ] 토스 SDK 연동 (필요한 경우): 결제 / 본인인증 / 알림
- [ ] 미니앱 권장 해상도 / 비율에서 레이아웃 깨짐 없음
- [ ] 토스 디자인 가이드의 톤(미니멀, 하이컨트라스트) 준수
- [ ] 외부 링크 진입 시 `Browser.open()`으로 인앱 브라우저 사용
- [ ] 백그라운드 진입 / 복귀 시 상태 복원
